## 1. 项目概述

### 1.1 项目背景

皮皮虾公司官网是皮皮虾前端开发公司的官方展示网站，旨在向潜在客户展示公司的专业能力、服务范围和团队实力。作为前端技术公司，该网站本身也是技术实力的直观体现。

### 1.2 项目目标

- **品牌展示**：建立专业、现代的企业形象
- **业务推广**：清晰展示服务内容和合作流程
- **客户转化**：提供便捷的联系方式和咨询入口
- **技术示范**：通过网站本身展示前端开发技术水平

---

## 2. 技术栈说明

本项目采用纯前端技术栈，无需后端服务器支持，可直接部署到任何静态托管平台。

| 技术 | 版本/说明 | 用途 |
|------|----------|------|
| **HTML5** | 语义化标签 | 页面结构构建 |
| **CSS3** | 原生 CSS | 样式设计、动画效果、响应式布局 |
| **JavaScript** | ES6+ | 交互逻辑、移动端菜单控制 |

### 2.1 核心技术特性

- **CSS 变量**：使用 `:root` 定义全局颜色变量，便于主题统一管理
- **Flexbox 布局**：用于导航栏、流程步骤等一维布局
- **Grid 布局**：用于组织架构、协作规范等二维布局
- **CSS 渐变**：Hero 区域和联系方式区域使用线性渐变背景
- **CSS 过渡动画**：按钮悬停、卡片悬浮等交互效果
- **媒体查询**：实现响应式断点适配

## 5. 响应式设计说明

### 5.1 断点设置

| 断点 | 屏幕宽度 | 适配策略 |
|------|----------|----------|
| **桌面端** | > 768px | 完整布局，横向展示 |
| **平板端** | ≤ 768px | 导航变为汉堡菜单，部分网格变为单列 |
| **手机端** | ≤ 480px | 进一步简化布局，减小内边距 |

### 5.2 适配策略详解

#### 导航栏适配
- **桌面端 (>768px)：** 横向导航链接完整显示
- **平板/手机端 (≤768px)：** 
  - 显示汉堡菜单按钮
  - 导航链接变为右侧滑出抽屉式菜单
  - 菜单宽度 280px，全屏高度

#### Hero 区域适配
- **桌面端：** 标题 3rem，内边距 160px/100px
- **平板端：** 标题 2rem，字体 1rem
- **手机端：** 内边距缩减为 120px/80px

#### 网格布局适配
- **公司简介统计：** 桌面端 3 列 → 平板端 1 列
- **组织架构卡片：** 自适应网格 → 手机端 1 列
- **协作规范卡片：** 最小 300px 自适应 → 手机端 1 列
- **联系方式：** 最小 250px 自适应 → 手机端自动堆叠

#### 流程步骤适配
- **桌面端：** 横向排列，带连接线
- **平板/手机端：** 纵向排列，隐藏连接线

### 5.3 响应式 CSS 代码位置

```css
/* 平板及手机端适配 */
@media (max-width: 768px) {
    /* 导航菜单、Hero、公司简介、流程步骤等适配 */
}

/* 小屏幕手机适配 */
@media (max-width: 480px) {
    /* 容器内边距、区块内边距、网格布局等适配 */
}
```

## 6. 使用说明

### 6.1 如何打开

**方法一：直接双击**
```
1. 在文件资源管理器中找到 pipiclaw-company-website.html
2. 双击文件，系统将使用默认浏览器打开
```

**方法二：浏览器打开**
```
1. 打开任意浏览器（Chrome、Edge、Firefox 等）
2. 将 HTML 文件拖拽到浏览器窗口
3. 或使用 Ctrl+O 快捷键选择文件打开
```

**方法三：本地服务器（推荐用于开发）**
```powershell
# 使用 PowerShell 启动本地服务器
cd C:\pipiclaw\workspace
python -m http.server 8080
# 然后访问 http://localhost:8080/pipiclaw-company-website.html
```

### 6.2 如何修改

#### 修改文字内容
1. 使用文本编辑器（VS Code、Notepad++ 等）打开 HTML 文件
2. 搜索需要修改的文字
3. 直接修改标签内的文本内容
4. 保存文件，刷新浏览器查看效果

**示例 - 修改公司标题：**
```html
<!-- 原代码 -->
<h1>皮皮虾前端开发公司</h1>

<!-- 修改后 -->
<h1>您的新公司名称</h1>
```

#### 修改颜色主题
在 `<style>` 标签顶部找到 `:root` 变量定义：

```css
:root {
    --primary-color: #2563eb;      /* 主色调（蓝色） */
    --primary-dark: #1d4ed8;       /* 主色深色变体 */
    --secondary-color: #64748b;    /* 辅助色 */
    --accent-color: #0ea5e9;       /* 强调色（青色） */
    /* ... 其他变量 */
}
```

修改这些变量值即可全局更新主题色。

#### 修改导航链接
找到 `<ul class="nav-links">` 部分：

```html
<ul class="nav-links" id="navLinks">
    <li><a href="#about">公司简介</a></li>
    <li><a href="#organization">组织架构</a></li>
    <!-- 添加/修改/删除导航项 -->
</ul>
```

#### 添加新模块
1. 在 `<body>` 中合适位置添加新的 `<section>` 标签
2. 复制现有 section 的结构作为参考
3. 添加对应的 CSS 样式
4. 在导航栏添加对应链接

## 7. 维护建议

### 7.1 日常维护

**每周检查：**
- [ ] 检查所有链接是否正常工作
- [ ] 验证联系方式是否准确
- [ ] 测试移动端显示效果

**每月检查：**
- [ ] 更新公司数据和统计信息
- [ ] 检查浏览器兼容性
- [ ] 审查内容准确性

### 7.2 性能优化建议

1. **图片优化**（如未来添加图片）
   - 使用 WebP 格式
   - 实施懒加载
   - 压缩图片大小

2. **代码优化**
   - 将 CSS 和 JS 提取到外部文件
   - 启用 Gzip 压缩
   - 使用 CDN 加速

3. **缓存策略**
   - 设置合适的 Cache-Control 头
   - 利用浏览器缓存

### 7.3 安全建议

1. **内容安全**
   - 避免内联 JavaScript（当前为演示目的保留）
   - 实施 CSP（内容安全策略）

2. **HTTPS**
   - 部署时务必使用 HTTPS
   - 配置 HSTS 头

3. **表单安全**（如未来添加表单）
   - 实施 CSRF 保护
   - 输入验证和转义
